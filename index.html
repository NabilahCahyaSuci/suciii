Multiple Inheritance & Class Modifiers

Polimorfisme adalah konsep di mana objek dari subclass bisa diperlakukan sebagai superclass, sehingga satu method dapat memiliki perilaku berbeda. Abstraksi adalah konsep di mana class tidak bisa diinstansiasi langsung (abstract class) dan digunakan untuk mendefinisikan method yang harus diimplementasikan oleh subclass.

Multiple Inheritance dengan Mixin

Multiple inheritance adalah pewarisan dari lebih dari satu sumber untuk menggabungkan perilaku atau properti. Di Dart, hal ini dilakukan menggunakan mixin, sehingga satu class bisa memiliki beberapa fungsi tanpa harus mewarisi banyak class secara langsung.

🔹 Studi Kasus Burung dan Bebek

Mixin adalah fitur untuk menambahkan method atau fungsi ke dalam class tanpa menggunakan pewarisan (extends). Digunakan untuk berbagi kode agar lebih fleksibel dan reusable.
// Nama : Nabilah Cahya Suci // NIM : 251410018 // Kelas : SI2A mixin CanFly { void fly() { print('Can fly!'); } } mixin CanSwim { void swim() { print('Can swim!'); } } class Bird with CanFly {} class Duck extends Bird with CanSwim {} void main() { Bird bird = Bird(); bird.fly(); Duck duck = Duck(); duck.fly(); duck.swim(); }

🔹 Studi Kasus Mixin dengan State

Mixin tidak hanya berisi method, tetapi juga bisa memiliki state (variabel). State ini bisa digunakan untuk menyimpan dan mengelola kondisi dalam class yang menggunakan mixin.
// Nama : Nabilah Cahya Suci // NIM : 251410018 // Kelas : SI2A mixin CanFly { bool _isFlying = false; bool get isFlying => _isFlying; void fly() { if (!_isFlying) { _isFlying = true; print('Started flying!'); } else { print('Already flying!'); } } void land() { if (_isFlying) { _isFlying = false; print('Landed!'); } else { print('Already on the ground!'); } } } mixin CanSwim { bool _isSwimming = false; bool get isSwimming => _isSwimming; void swim() { if (!_isSwimming) { _isSwimming = true; print('Started swimming!'); } else { print('Already swimming!'); } } void stopSwim() { if (_isSwimming) { _isSwimming = false; print('Stop swimming!'); } else { print('Already stopped!'); } } } class Bird with CanFly {} class Duck extends Bird with CanSwim {} class Fish with CanSwim {} class Airplane with CanFly {} void main() { Bird bird = Bird(); bird.fly(); bird.land(); print('----------------'); Duck duck = Duck(); duck.fly(); duck.swim(); duck.stopSwim(); print('----------------'); Fish fish = Fish(); fish.swim(); fish.stopSwim(); print('----------------'); Airplane airplane = Airplane(); airplane.fly(); airplane.land(); }

🔹 Pengantar Class Modifiers

Class modifiers adalah keyword (sealed, base, final) untuk mengatur bagaimana class digunakan atau diwariskan. Tujuannya agar kode lebih aman, terstruktur, dan terkontrol.

🔹 Sealed Class

Sealed class adalah class yang hanya bisa diwariskan di dalam file yang sama. Digunakan untuk membatasi turunan agar lebih aman dan terkontrol.
// Nama : Nabilah Cahya Suci // NIM : 251410018 // Kelas : SI2A sealed class ApiResponse { const ApiResponse(); } class Success extends ApiResponse { final String message; const Success(this.message); } class ErrorResponse extends ApiResponse { final String errorMessage; const ErrorResponse(this.errorMessage); } void handle(ApiResponse response) { if (response is Success) { print("Success: ${response.message}"); } else if (response is ErrorResponse) { print("Error: ${response.errorMessage}"); } } void main() { handle(const Success("Data berhasil")); handle(const ErrorResponse("Terjadi kesalahan")); }

🔹 Keuntungan Sealed Class

Keuntungan dari sealed class adalah dapat membatasi pewarisan hanya dalam satu file, sehingga lebih aman dan mudah dikontrol. Selain itu, semua kemungkinan turunan class sudah diketahui, sehingga membantu mengurangi kesalahan saat program dijalankan dan membuat kode lebih terstruktur.
// Nama : Nabilah Cahya Suci // NIM : 251410018 // Kelas : SI2A void handleResponse(ApiResponse response) { switch (response) { case Success(): print("Success: ${(response as Success).message}"); break; case Error(): print("Error: ${(response as Error).errorMessage}"); break; } } void main() { handleResponse(Success("Data berhasil")); handleResponse(Error("Terjadi kesalahan")); } sealed class ApiResponse {} class Success extends ApiResponse { final String message; Success(this.message); } class Error extends ApiResponse { final String errorMessage; Error(this.errorMessage); }

🔹 Base Class

Base class adalah class yang bisa diwariskan menggunakan extends, tetapi tidak bisa di-implement. Tujuannya agar pewarisan tetap terkontrol dan hanya menggunakan satu cara, sehingga kode lebih konsisten dan terstruktur.
// Nama : Nabilah Cahya Suci // NIM : 251410018 // Kelas : SI2A base class Animal { void speak() { print("Hewan bersuara..."); } } base class Dog extends Animal { @override void speak() { print("woof woff!"); } } void main() { Animal myDog = Dog(); myDog.speak(); }

🔹 Kapan Menggunakan Base Class

penggunaan base class, misalnya class Animal yang diturunkan menjadi Dog dan Cat menggunakan extends. Setiap turunan bisa memiliki perilaku berbeda, tetapi tetap mengikuti struktur dari base class, sehingga kode lebih rapi dan terorganisir.
// Nama : Nabilah Cahya Suci // NIM : 251410018 // Kelas : SI2A base class Animal { void speak() { print("Hewan bersuara..."); } } base class Dog extends Animal { @override void speak() { print("Woof!"); } } void main() { Animal animal = Dog(); animal.speak(); }

🔹 Final Class

Final class adalah class yang tidak bisa diwariskan lagi (tidak bisa di-extend atau di-implement). Tujuannya untuk “mengunci” class agar tidak diubah atau diturunkan, sehingga kode lebih aman dan stabil.
// Nama : Nabilah Cahya Suci // NIM : 251410018 // Kelas : SI2A final class Settings { final String theme; final int fontSize; Settings({required this.theme, required this.fontSize}); void showSettings() { print("Theme: $theme, Font Size: $fontSize"); } } class CustomSettings { final Settings settings; CustomSettings(String theme, int fontSize) : settings = Settings(theme: theme, fontSize: fontSize); void showSettings() { settings.showSettings(); } } void main() { Settings settings = Settings(theme: "Dark", fontSize: 14); settings.showSettings(); CustomSettings custom = CustomSettings("Dark", 14); custom.showSettings(); }

🔹 Perbedaan Class Modifiers

Class modifiers adalah keyword yang digunakan untuk mengatur akses dan cara penggunaan suatu class. Modifier seperti public memungkinkan class diakses dari mana saja, sedangkan private (ditandai dengan _) hanya bisa diakses dalam file yang sama. Selain itu, ada modifier yang mengatur perilaku class, seperti abstract yang tidak bisa dibuat objek langsung dan hanya untuk diwariskan, final yang tidak bisa diturunkan lagi, base yang membatasi pewarisan, interface yang hanya sebagai kontrak tanpa implementasi langsung, serta sealed yang membatasi pewarisan hanya dalam file yang sama. Jadi, secara sederhana, class modifiers menentukan siapa yang bisa mengakses class dan bagaimana class tersebut dapat digunakan atau diturunkan.

🔹 Praktik 1: Polimorfisme dengan List

Praktik ini menggunakan sealed class dan pattern matching untuk menangani berbagai kondisi data secara aman dan terstruktur.
// Nama : Nabilah Cahya Suci // NIM : 251410018 // Kelas : SI2A sealed class Status { const Status(); } class Loading extends Status { const Loading(); } class Loaded extends Status { final String data; const Loaded(this.data); } class Failed extends Status { final String error; const Failed(this.error); } void handleStatus(Status status) { switch (status) { case Loading(): print('Loading...'); case Loaded(:final data): print('Data: $data'); case Failed(:final error): print('Error: $error'); } } void main() { handleStatus(const Loading()); handleStatus(const Loaded("Data berhasil")); handleStatus(const Failed("Gagal")); }

🔹 Studi Kasus: Sistem State Managements

Studi kasus ini membahas penggunaan state management untuk mengelola berbagai kondisi (state) dalam aplikasi, seperti loading, success, dan error. Biasanya menggunakan sealed class agar semua kemungkinan state terdefinisi dengan jelas. Dengan cara ini, pengelolaan alur program menjadi lebih terstruktur, mudah dikontrol, dan mengurangi kesalahan.
// Nama : Nabilah Cahya Suci // NIM : 251410018 // Kelas : SI2A sealed class AppState {} class LoadingState extends AppState {} class SuccessState extends AppState { final List items; SuccessState(this.items); } class ErrorState extends AppState { final String message; ErrorState(this.message); } void showLoading() { print('Loading...'); } void showItems(List items) { print('Success! Items:'); for (int i = 0; i < items.length; i++) { print('${i + 1}. ${items[i]}'); } } void showError(String message) { print('Error: $message'); } void updateUI(AppState state) { switch (state) { case LoadingState(): showLoading(); case SuccessState(:final items): showItems(items); case ErrorState(:final message): showError(message); } } void main() { updateUI(LoadingState()); updateUI(SuccessState(["Item 1", "Item 2"])); updateUI(ErrorState("Terjadi error")); }

🔹 Ringkasan & Base Practices

Materi ini menekankan penggunaan mixin dan class modifiers (sealed, base, final) untuk membuat kode lebih rapi dan aman. Best practices-nya adalah menggunakan sealed untuk membatasi state, base untuk pewarisan terkontrol, dan final untuk mengunci class, serta memanfaatkan mixin agar kode lebih fleksibel dan dapat digunakan ulang.

🔹 Bonus 1 Base Class Repository

Base class repository adalah pola penggunaan base class sebagai dasar untuk mengelola data (misalnya dari API atau database). Class ini berisi fungsi umum seperti mengambil, menyimpan, atau menghapus data, lalu diwariskan ke class lain agar kode lebih terstruktur, konsisten, dan mudah digunakan ulang.
// Nama : Nabilah Cahya Suci // NIM : 251410018 // Kelas : SI2A base class BarangRepository { void create(String barang) { print("Tambah barang: $barang"); } void read() { print("Menampilkan semua barang..."); } void update(int id, String barang) { print("Barang ke-$id diubah jadi $barang"); } void delete(int id) { print("Barang ke-$id dihapus"); } } // subclass beda dari temen base class Gudang extends BarangRepository { @override void read() { print("Daftar barang di gudang:"); } } void main() { print("=== SISTEM GUDANG ==="); Gudang gudang = Gudang(); gudang.create("Sepatu"); gudang.read(); gudang.update(1, "Sepatu Baru"); gudang.delete(1); }

🔹 Bonus 2 Final Class Mathutlis

Final class MathUtils adalah class utilitas untuk operasi matematika yang tidak bisa diwariskan. Class ini berisi fungsi-fungsi umum seperti perhitungan angka, sehingga lebih aman, stabil, dan digunakan langsung tanpa diubah.
// Nama : Nabilah Cahya Suci // NIM : 251410018 // Kelas : SI2A final class Hitung { static int tambah(int a, int b) => a + b; static int kurang(int a, int b) => a - b; static int kali(int a, int b) => a * b; static double bagi(double a, double b) => a / b; } void main() { print("=== PROGRAM HITUNG ==="); print("Penjumlahan : ${Hitung.tambah(15, 5)}"); print("Pengurangan : ${Hitung.kurang(15, 5)}"); print("Perkalian : ${Hitung.kali(15, 5)}"); print("Pembagian : ${Hitung.bagi(15, 5)}"); }
