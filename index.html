Pengertian Multiple Inheritance & Class Modifiers (Sealed, Base, Final)

Multiple Inheritance & Class Modifiers adalah konsep untuk mengatur pewarisan class. Multiple inheritance memungkinkan class punya banyak perilaku (mixin), sedangkan sealed, base, dan final membatasi penggunaan class agar lebih aman.

🔹 Polimorfisme & Abstraksi

Polimorfisme: Objek subclass sebagai superclass Abstract class: tidak bisa diinstansiasi, bisa punya method abstract/concrete Interface: implicit di Dart, gunakan implements Contoh cepat:
abstract class Bentuk { double hitungLuas(); } class Persegi implements Bentuk { @override double hitungLuas() => sisi * sisi; }

🔹 Multiple Inheritance dengan Mixin

Multiple Inheritance dengan Mixin adalah cara agar satu class memiliki banyak perilaku dengan menambahkan mixin, tanpa mewarisi banyak class langsung, sehingga lebih fleksibel dan bisa digunakan ulang.

🔹 Studi Kasus: Burung dan Bebek

Inheritance: extends keyword
// Nama : Nabilah Cahya Suci // NIM : 251410018 // Kelas : SI2A mixin CanFly { void fly() { print('Can fly!'); } } mixin CanSwim { void swim() { print('Can swim!'); } } class Bird with CanFly {} class Duck extends Bird with CanSwim {} void main() { Bird bird = Bird(); bird.fly(); Duck duck = Duck(); duck.fly(); duck.swim(); }

🔹Studi Kasus: mixin dengan state

Referensi parent bisa menampung objek child. Method yang dipanggil sesuai objek asli (runtime).
// Nama : Nabilah cahya suci // NIM : 251410018 // Kelas : SI2A mixin CanFly { bool _isFlying = false; bool get isFlying => _isFlying; void fly() { if (!_isFlying) { _isFlying = true; print('Started flying!'); } else { print('Already flying!'); } } void land() { if (_isFlying) { _isFlying = false; print('Landed!'); } else { print('Already on the ground!'); } } } mixin CanSwim { bool _isSwimming = false; bool get isSwimming => _isSwimming; void swim() { if (!_isSwimming) { _isSwimming = true; print('Started swimming!'); } else { print('Already swimming!'); } } void stopSwim() { if (_isSwimming) { _isSwimming = false; print('Stop swimming!'); } else { print('Already stopped!'); } } } class Bird with CanFly {} class Duck extends Bird with CanSwim {} class Fish with CanSwim {} class Airplane with CanFly {} void main() { Bird bird = Bird(); bird.fly(); bird.land(); print('----------------'); Duck duck = Duck(); duck.fly(); duck.swim(); duck.stopSwim(); print('----------------'); Fish fish = Fish(); fish.swim(); fish.stopSwim(); print('----------------'); Airplane airplane = Airplane(); airplane.fly(); airplane.land(); }

🔹 Pengantar Class Modifiers

Apa itu? Keyword untuk mengontrol bagaimana class dapat digunakan Tujuan: i. Kontrol inheritance/implementasi ii. Jaminan keamanan tipe iii. Desain API yang lebih ekspresif Macam-macam: sealed , base , final , interface (Dart 3.0+)

🔹 Sealed Class

Fitur Tidak bisa id-ectend atau di-implement dar luar library Membatasi hierarki tipe yang diketahui saat compile-time Perfect untuk pattern matching
//Nama: Nabilah cahya suci //Nim: 251410018 //Kelas: SI2A sealed class ApiResponse { const ApiResponse(); } class Success extends ApiResponse { final String message; const Success(this.message); } class ErrorResponse extends ApiResponse { final String errorMessage; const ErrorResponse(this.errorMessage); } void handle(ApiResponse response) { if (response is Success) { print("Success: ${response.message}"); } else if (response is ErrorResponse) { print("Error: ${response.errorMessage}"); } } void main() { handle(const Success("Data berhasil")); handle(const ErrorResponse("Terjadi kesalahan")); }

🔹 Keuntungan sealed class

Keuntungan Exhaustive checking: Compiler tahu semua kemungkinan subclass Pattern matching: Switch case yang aman Documentation: Jelas hierarki terbatas
// Nama : Nabilah Cahya Suci // NIM : 251410018 // Kelas : SI2A abstract class Kendaraan { String? merk; void start(); // abstract void stop() { print('Kendaraan berhenti'); } } class Mobil extends Kendaraan { @override void start() { print('Mobil $merk menyala'); } } void main() { Mobil m = Mobil(); m.merk = "Toyota"; m.start(); m.stop(); }

🔹 Base class

Fitur: Bisa di-extend, tapi tidak bisa di-implement Memastikan pewarisan dengan extends , bukan implements Berguna untuk library yang ingin mengontrol inheritance
// Nama : Nabilah Cahya Suci // NIM : 251410018 // Kelas : SI2A abstract class Kendaraan { String? merk; void start(); // abstract void stop() { print('Kendaraan berhenti'); } } class Mobil extends Kendaraan { @override void start() { print('Mobil $merk menyala'); } } void main() { Mobil m = Mobil(); m.merk = "Toyota"; m.start(); m.stop(); }

🔹 kapan menggunakan Base class

saat ingin membuat hierarki yang hanya bisa di-extend saat ingin menjaga struktur class tetap konsisten saat ingin Mencegah implements yang tidak sesuai saat membutuhkan kontrol penuh pada class turunan
// Nama : Nabilah Cahya Suci // NIM : 251410018 // Kelas : SI2A abstract class AlatMusik { void mainkan(); void info() { print('Ini adalah alat musik'); } } class Gitar extends AlatMusik { @override void mainkan() { print('Jreng jreng...'); } } void main() { Gitar g = Gitar(); g.info(); g.mainkan(); }

🔹 Final Class

Fitur: Bisa di-extend, tapi tidak bisa di-implement Memastikan pewarisan dengan extends , bukan implements Berguna untuk library yang ingin mengontrol inheritance
// Nama : Nabilah Cahya Suci // NIM : 251410018 // Kelas : SI2A interface class Flyable { void terbang() => print('Terbang...'); } class Bird implements Flyable { @override void terbang() => print('Burung terbang tinggi'); } class Plane implements Flyable { @override void terbang() => print('Pesawat terbang cepat'); } void main() { Bird b = Bird(); Plane p = Plane(); b.terbang(); p.terbang(); }

🔹 Perbedaan Class Modifiers

Sealed: Hanya bisa di-extend di dalam file yang sama; tidak bisa di-extend atau di-implement di luar library. base: Bisa di-extend di luar library, tetapi tidak bisa di-implement (hanya mendukung pewarisan). final: Bisa di-extend atau di-implement di dalam file yang sama, namun keduanya dilarang dilakukan di luar library. default:Tidak memiliki batasan; bebas untuk di-extend maupun di-implement baik di dalam maupun di luar library.

🔹 Praktik 1-Sealed class dengan pattern matching

Sealed class memungkinkan semua subclass diketahui dalam satu file sehingga dengan pattern matching setiap kondisi bisa diperiksa dan ditangani dengan jelas.
// Nama : Nabilah Cahya Suci // NIM : 251410018 // Kelas : SI2A abstract class MetodePembayaran { void bayar(double jumlah); } class KartuKredit implements MetodePembayaran { @override void bayar(double jumlah) { print('Bayar $jumlah dengan Kartu Kredit'); } } class EWallet implements MetodePembayaran { @override void bayar(double jumlah) { print('Bayar $jumlah dengan E-Wallet'); } } class Tunai implements MetodePembayaran { @override void bayar(double jumlah) { print('Bayar $jumlah dengan Tunai'); } } void main() { List metode = [ KartuKredit(), EWallet(), Tunai() ]; for (var m in metode) { m.bayar(100000); } }

🔹 Studi Kasus: Sistem State Management

State Management adalah cara mengatur kondisi aplikasi (loading, success, error) dengan sealed class agar tiap state jelas.
// Nama : Nabilah Cahya Suci // NIM : 251410018 // Kelas : SI2A abstract class Bentuk { double hitungLuas(); double hitungKeliling(); void info() { print('Luas: ${hitungLuas()}, Keliling: ${hitungKeliling()}'); } } class Persegi implements Bentuk { double sisi; Persegi(this.sisi); @override double hitungLuas() => sisi * sisi; @override double hitungKeliling() => 4 * sisi; @override void info() { print('Persegi sisi $sisi → Luas ${hitungLuas()} | Keliling ${hitungKeliling()}'); } } class Lingkaran implements Bentuk { double radius; Lingkaran(this.radius); @override double hitungLuas() => 3.14 * radius * radius; @override double hitungKeliling() => 2 * 3.14 * radius; @override void info() { print('Lingkaran radius $radius → Luas ${hitungLuas()} | Keliling ${hitungKeliling()}'); } } void main() { Bentuk b1 = Persegi(4); Bentuk b2 = Lingkaran(7); b1.info(); b2.info(); }

Ringkasan

Ringkasan Sealed: Hierarki tertutup & aman untuk pattern matching. Base: Hanya bisa di-extend (tutup akses implement). Final: Segel total (tidak bisa di-extend atau di-implement). Quick Guide: Sealed: rightarrow Untuk State atau Result terbatas. Base: rightarrow Untuk Framework atau Base Class. Final: rightarrow Untuk Utility atau Constant Class.
